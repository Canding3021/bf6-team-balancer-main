# 版本更新说明

## v2.1.0 - 2026-06-02

### 新增

- **双源 API 查询**：引入 joarchy.com（BF6 Tracker API）作为第二数据源，与 gametools.network 互补
- **KPM 修正**：优先使用 joarchy 的 KPM（去除了菜单/挂机时间，更接近真实战斗效率）
- **joarchy 兜底查询**：gametools 查不到的玩家，自动用 joarchy 按 name 查询补充
- **偏移系数冻结**：API 覆盖率 ≥ 80% 时自动冻结偏移系数，未查到的玩家直接使用 Excel 原始数据，避免小样本偏移不准

### 变更

- **偏移系数计算**：仅基于 gametools 数据计算，不再混入 joarchy 的 KPM 值（joarchy KPM 口径不同，混入会导致偏移虚高）
- **查询进度显示**：区分 gametools / joarchy / joarchy兜底 三个阶段的状态文字
- **第二次查询**：断开旧 worker 信号连接，避免重复查询时 UI 不刷新

### 优化

- joarchy 查询自带主备节点容错（UK 主节点 → US 备用节点）
- userId 字段从 gametools 返回值中提取，作为 joarchy 的稳定 identifier 查询

## v2.0.0 - 2026-06-01

### 新增

- **API 自动查询**：通过 gametools.network 接口，用 EAID 自动查询玩家真实 KD/KPM，无需手动填入准确数据
- **动态偏移系数**：根据查到的玩家数据自动计算偏移系数，替代固定值 1.313；查不到的玩家用 Excel 数据 × 动态偏移
- **查询进度页**：新增 API 查询加载页，转圈动画 + 进度条 + 实时统计（查到/未找到/无EAID）
- 查询完成后显示 ✓ 确认标记，自动跳转下一步

### 变更

- **Excel 格式扩展**：从 3 列（昵称/KD/KPM）改为 4 列（昵称/KD/KPM/EAID），EAID 为必填
- **移除手动偏移控件**：KPM 偏移系数不再支持手动调整，改为 API 动态计算
- **候补队伍改版**：从底部挤压改为与两阵营并排显示（三列布局），去掉序号列
- **主题配色统一**：所有硬编码颜色改为从主题字典读取，切换主题时正确刷新全部页面
- **黑白主题可读性**：按钮和步骤指示器文字颜色跟随主题，不再白底白字
- 步骤指示器从 6 步扩展为 7 步（新增 API 查询步骤）

### 优化

- 移除未使用的 `import math`
- 移除未使用的 `format_record_text` 导入
- 精确异常捕获（`OSError, ValueError` 替代裸 `Exception`）
- 空数据导入时弹出警告提示
- Spinner 动画按需启动，不在程序启动时就开始转

### 算法

- `allocate_teams` 修复 `max_per_team` 翻倍计算错误
- `Player` 分数缓存改为按游戏模式分别缓存（修复切换模式后分数不变的 bug）
- `Player.__hash__/__eq__` 改为基于名称（修复去重失效）
- `compute_balance_report` 增加 None 值保护

### 新增文件

- `api_query.py`：API 查询模块，8 路并发，偏移计算
- `test_api.py`：API 可用性验证脚本

## v1.1.0 - 2026-06-01

### 新增

- **随机分配模式**：新增 🎲 随机分配选项，纯随机打乱玩家分配阵营，适合娱乐局或测试
- **分配模式选择页**：导入玩家后新增一步，让用户选择「均衡分配」或「随机分配」
- 随机模式自动跳过游戏模式选择页（步骤指示器显示删除线标记）
- 结果页随机模式显示「随机分配」标签，隐藏均衡性告警
- **文字导出**：结果页「复制结果」按钮，格式化纯文本写入剪贴板，方便粘贴到 QQ/Discord
- **历史记录**：自动保存最近 5 次分队结果，独立历史页可浏览记录列表并点击查看详情，存储于 `%USERPROFILE%\Documents\BF6TeamBalancer\history.json`
- **操作页**：结果页之后新增操作汇总页，提供「重新导入」「重新分配」「退出」三个按钮

### 优化

- UI 步骤指示器从 4 步扩展为 6 步（导入 → 分配模式 → 游戏模式 → 自定义小队 → 结果 → 操作）
- 随机模式下返回上一步自动跳过游戏模式页
- 自定义小队绑定在随机模式下仍然生效（不能拆散）
- 「重新导入」可清空数据回到起点，无需重启程序
- 「重新分配」保留玩家数据，回到分配模式选择

### 算法

- 新增 `random_allocate()` 函数：random.shuffle 打乱 → AABB 轮流分配 → 4 人编队 → 候补抽取
- 均衡模式算法无变动，原有行为完全保留
